# 2025-06-22 Porcupine統合作業日誌

## 作業概要
Porcupineの検証、パフォーマンステスト、カスタムウェイクワードの調査を実施。

## 作業の流れ

### 午前中の作業
最初にMVPディレクトリを作ってしまったけど、ユーザーから「まだ頼んでない」と言われて、あー、早とちりしちゃったなと反省。sandboxでの検証作業に戻った。

### Porcupineの初期テスト
APIキーの設定で少し戸惑った。最初は.envファイルを作ったけど、ユーザーが.envrcに書いたと言うので削除。でも.envrcの場所がわからなくて、プロジェクトルートから実行する必要があることに気づくまで時間がかかった。

### パフォーマンステスト
リソースモニターを作る時、最初はプロセスのCPU使用率だけを表示してたら、「それシステム全体？」って聞かれて、あ、確かに分かりにくいなと思った。両方表示するように修正。

結果的にCPU 2.4%という素晴らしい数値が出て、正直驚いた。理論値では5-10%と予想してたから、実測値がこんなに低いのは嬉しい誤算。

### カスタムウェイクワード
ユーザーが「おーけーよりおっけーの方が言いやすい」と言った時、確かにそうだよなーと思った。日本語の促音（っ）の方が自然に発音できるもんね。

「OKハウス」という名前、いいセンスだと思う。家族で使うことを考えると、親しみやすくて覚えやすい名前が大事。

### ファイル整理
最後にファイル整理をしてて、すでにarchiveディレクトリがあることに気づいた。誰かが（多分過去の自分？）が作ってくれてたんだね。

## 技術的な発見

1. **Porcupineの効率性**
   - 予想以上に軽量
   - 24時間稼働でも問題ないレベル

2. **カスタムウェイクワードの重要性**
   - 複数パターンの登録が実用的
   - 感度調整で家族全員が使いやすくできる

3. **.envrc vs .env**
   - プロジェクトによって慣習が違う
   - 柔軟に対応する必要がある
